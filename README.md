# POK Web Manager

A modern, responsive Web Dashboard and Discord Bot companion for managing ARK Survival Ascended dedicated servers, fully layout-compatible with the upstream `POK-manager.sh` directory conventions.

## 🚀 Key Features

*   **Real-time Dashboard UI**: Start, stop, restart, and monitor live container metrics (CPU/Memory usage charts) directly from your web browser.
*   **Discord Bot Integration**: Relays in-game server chat to instance-specific Discord channels, lets admins run commands (like `/start` or `/stop`) directly from Discord, and posts automated lifecycle announcements.
*   **Robust Automatic Updates**: Periodically polls SteamCMD in the background, downloads updates to a temporary folder while servers remain online, alerts players with countdown warnings, and swaps files atomically in milliseconds.
*   **Player-Aware Countdown Bypass**: Automatically checks server occupancy every 10 seconds during update countdowns; if all players log out early, the manager immediately saves and restarts the server to reduce downtime.
*   **Stale Manifest & Lock Recovery**: Automatically writes clean basic appmanifest files before runs to bypass SteamCMD `Access Denied` loops and `Missing configuration` errors.
*   **Live Log Streaming**: Streams container console outputs in real-time to the dashboard console using Server-Sent Events (SSE).

## 🔒 Security Warning

> [!WARNING]
> **Container Socket Exposure**: This Web Manager requires mounting `/var/run/docker.sock` to manage your server containers. Anyone with admin access to the manager dashboard effectively holds root control over your Docker daemon.
> 
> To secure your deployment:
> 1. **Change the Default Password**: Always override the default `POK_ADMIN_PASSWORD=admin` in your `docker-compose.yaml` with a strong password.
> 2. **Restrict Port Access**: Do **NOT** expose port `8084` directly to the public internet. Use a secure VPN (such as WireGuard or Tailscale) or set up a reverse proxy (e.g., Nginx, Traefik) with SSL/TLS and IP Whitelisting.
> 3. **Firewall Rules**: Ensure your firewall restricts incoming TCP traffic on port `8084` only to trusted local network addresses.

## 📂 Layout Prerequisites

The Web Manager expects to be run in (or pointed to) a directory organized with the standard `POK-manager` conventions:

```text
/path/to/your/POK-manager/
├── docker-compose.yaml            # This manager compose configuration
├── defaults/
│   ├── GameUserSettings.ini       # Default game settings
│   └── Game.ini                   # Default game configurations
├── config/
│   └── POK-manager/               # Instance credentials and system variables
├── Instance_island_pvp/           # Example server instance folder
│   ├── Saved/                     # Game saved files
│   └── docker-compose-island_pvp.yaml  # Autogenerated container configuration
└── ServerFiles/
    └── arkserver/                 # Shared server binary directory
```

## 🛠️ Quick Start Guide

### 1. Configure Docker Compose
Create or edit the `docker-compose.yaml` file in your root folder. Update the host paths to match your system directory:

```yaml
version: '3.8'

services:
  pok-manager:
    image: sselph/pok-manager-ui:main
    container_name: pok_manager
    restart: unless-stopped
    extra_hosts:
      - "host.docker.internal:host-gateway"
    ports:
      - "8084:8084"
    environment:
      # Absolute path of your POK-manager folder on the host machine (CRITICAL for volume mapping)
      - HOST_BASE_DIR=/path/to/your/POK-manager
      # Web panel admin password (username: admin)
      - POK_ADMIN_PASSWORD=admin
      - PUID=7777
      - PGID=7777
      
      # Default settings applied automatically when creating new server instances via the UI
      - DEFAULT_CLUSTER_ID=cluster
      - DEFAULT_SERVER_PASSWORD=
      - DEFAULT_ADMIN_PASSWORD=myadminpassword
      - DEFAULT_MOD_IDS=
      - DEFAULT_PASSIVE_MODS=
      - DEFAULT_CUSTOM_SERVER_ARGS=
      
      # Optional: Discord bot configurations
      - DISCORD_TOKEN=
      - DISCORD_GUILD_ID=
      - DISCORD_CHANNEL_ID=
    volumes:
      - /var/run/docker.sock:/var/run/docker.sock
      - /path/to/your/POK-manager:/app/workspace
```

### 2. Build & Launch
Deploy the manager stack:
```bash
docker compose up --build -d
```
Once started, access the Web UI dashboard at `http://<your-host-ip>:8084` and log in with user `admin` and your configured password.

## 🤖 Discord Bot Setup

To enable RCON chat relaying, status notifications, and command triggers from Discord:

1.  Go to the [Discord Developer Portal](https://discord.com/developers/applications) and create a New Application.
2.  Navigate to the **Bot** tab, generate a token, and paste it into the `DISCORD_TOKEN` environment variable.
3.  Ensure **Privileged Gateway Intents** (specifically **Server Members Intent** and **Message Content Intent**) are toggled **ON** in the developer portal.
4.  Copy your Discord Server (Guild) ID and input it into `DISCORD_GUILD_ID`.
5.  To configure channel-specific game chat relays, open the instance configuration inside the Web UI and input the target **Discord Channel ID** in the server settings panel.
6.  *(Optional)* Provide a channel ID in `DISCORD_CHANNEL_ID` to receive manager-wide update and check announcements.

## 🔧 Troubleshooting

### RCON Connection Failures
The Web Manager runs inside a Docker container and communicates with your game servers via RCON.
*   By default, it attempts to connect using `host.docker.internal`.
*   If connections time out or fail, set the `RCON_HOST` environment variable in your `docker-compose.yaml` directly to your host's local network IP (e.g., `192.168.1.10`).

### SteamCMD update loops (`Missing configuration`)
If SteamCMD halts during automated updates claiming a missing configuration or returns `0x6` error codes:
*   Ensure that no host mounts are masking `/home/pok/.steam/steam/steamapps/compatdata` inside the updater container. The Web Manager automatically runs updates using clean basic manifests to prevent this.
