# POK Web Manager

A modern, responsive Web Dashboard and Discord Bot companion for managing ARK Survival Ascended dedicated servers, fully layout-compatible with the upstream `POK-manager.sh` directory conventions.

## 🚀 Key Features

*   **Real-time Dashboard UI**: Start, stop, restart, and monitor live container metrics (CPU/RAM usage charts) directly from your web browser. Includes manual override controls ("Force Reboot All", "Fresh Install").
*   **Smart Major vs. Minor Updates**: Automatically classifies SteamCMD updates via offline version probing (`MAP_NAME=InvalidMapName`). Minor updates swap instantly and restart instances 1-by-1 as players log off; major updates run a cluster countdown with RCON warnings.
*   **OpenZFS Reflink Cloning**: Stages server updates in milliseconds using copy-on-write (`cp -a --reflink=always`) and routes SteamCMD temporary downloads to `$TEMP_DOWNLOAD_ROOT` to eliminate SSD wear.
*   **Discord Slash Commands**: Fully controls server lifecycle, RCON chat relays, world saves, status checks, and forced updates directly from Discord.
*   **Player-Aware Countdown Bypass**: Automatically checks player occupancy during countdowns; if all players log out early, the manager saves and restarts immediately.
*   **Live Log Streaming**: Real-time container console streaming via Server-Sent Events (SSE).

## 🔒 Security Warning

> [!WARNING]
> **Container Socket Exposure**: This Web Manager requires mounting `/var/run/docker.sock` to manage your server containers. Anyone with admin access to the manager dashboard effectively holds root control over your Docker daemon.
> 
> To secure your deployment:
> 1. **Change the Default Password**: Always override the default `POK_ADMIN_PASSWORD=admin` in your `docker-compose.yaml` with a strong password.
> 2. **Restrict Port Access**: Do **NOT** expose port `8084` directly to the public internet. Use a secure VPN (such as WireGuard or Tailscale) or set up a reverse proxy (e.g., Nginx, Traefik) with SSL/TLS and IP Whitelisting.
> 3. **Firewall Rules**: Ensure your firewall restricts incoming TCP traffic on port `8084` only to trusted local network addresses.

## 📂 Layout Prerequisites

The Web Manager expects to be run in (or pointed to) a directory organized with the standard `POK-manager` conventions:

```text
/path/to/your/POK-manager/
├── docker-compose.yaml            # This manager compose configuration
├── defaults/
│   ├── GameUserSettings.ini       # Default game settings
│   └── Game.ini                   # Default game configurations
├── config/
│   └── POK-manager/               # Instance credentials and system variables
├── Instance_island_pvp/           # Example server instance folder
│   ├── Saved/                     # Game saved files
│   └── docker-compose-island_pvp.yaml  # Autogenerated container configuration
└── ServerFiles/
    ├── arkserver/                 # Shared live server binary directory
    └── arkserver_temp/            # Temporary staged update directory (ZFS reflink)
```

## 🛠️ Quick Start Guide

### 1. Configure Docker Compose
Create or edit the `docker-compose.yaml` file in your root folder. Update the host paths to match your system directory:

```yaml
version: '3.8'

services:
  pok-manager:
    image: sselph/pok-manager-ui:main
    container_name: pok_manager
    restart: unless-stopped
    extra_hosts:
      - "host.docker.internal:host-gateway"
    ports:
      - "8084:8084"
    environment:
      # Absolute path of your POK-manager folder on the host machine (CRITICAL for volume mapping)
      - HOST_BASE_DIR=/path/to/your/POK-manager
      # Web panel admin password (username: admin)
      - POK_ADMIN_PASSWORD=admin
      - PUID=7777
      - PGID=7777
      
      # Default settings applied automatically when creating new server instances via the UI
      - DEFAULT_CLUSTER_ID=cluster
      - DEFAULT_SERVER_PASSWORD=
      - DEFAULT_ADMIN_PASSWORD=myadminpassword
      - DEFAULT_MOD_IDS=
      - DEFAULT_PASSIVE_MODS=
      - DEFAULT_CUSTOM_SERVER_ARGS=
      
      # Optional: Custom temporary download path for SteamCMD staging
      - TEMP_DOWNLOAD_ROOT=/path/to/your/POK-manager/ServerFiles/steamapps/temp
      
      # Optional: Discord bot configurations
      - DISCORD_TOKEN=
      - DISCORD_GUILD_ID=
      - DISCORD_CHANNEL_ID=
    volumes:
      - /var/run/docker.sock:/var/run/docker.sock
      - /path/to/your/POK-manager:/app/workspace
```

### 2. Build & Launch
Deploy the manager stack:
```bash
docker compose up --build -d
```
Once started, access the Web UI dashboard at `http://<your-host-ip>:8084` and log in with user `admin` and your configured password.

## 🤖 Discord Bot & Slash Commands

The Web Manager registers native Discord Slash Commands on boot:

### 🌐 Public Commands (Available to All Users)
*   `/status`: Displays cluster status (`ONLINE`, `STARTING`, `UNHEALTHY`, or grouped `OFFLINE`), player count, RAM usage, ARK version, and pending update badges.
*   `/listplayers server:<instance>`: Lists all players currently connected to the specified server instance.
*   `/chat msg:<message>`: Sends a message to in-game chat for the server mapped to the current Discord channel.

### 🛡️ Admin Commands (Requires `Manage Server` Permission)
*   `/start server:<instance>`: Starts the specified server instance container.
*   `/stop server:<instance>`: Stops the specified server instance container cleanly.
*   `/restart server:<instance>`: Restarts the specified server instance container cleanly.
*   `/update server:<instance>`: Pulls the latest container image and restarts the server instance.
*   `/saveworld server:<instance>`: Executes an RCON `SaveWorld` command to force an immediate save.
*   `/forceupdate [fresh_install:bool] [force_reboot:bool]`: Triggers global ARK game update orchestration.
    *   `fresh_install`: Wipes server files and re-downloads game binaries from scratch.
    *   `force_reboot`: Forces immediate reboot of active servers even if the update is minor.

### ⚙️ Discord Bot Setup
1. Go to the [Discord Developer Portal](https://discord.com/developers/applications) and create an Application.
2. Under **Bot**, generate a token and paste it into `DISCORD_TOKEN`. Enable **Message Content Intent**.
3. Set your Discord Server (Guild) ID in `DISCORD_GUILD_ID`.
4. Configure channel-specific game chat relays by setting the **Discord Channel ID** in instance settings via the Web UI.
5. *(Optional)* Set `DISCORD_CHANNEL_ID` to receive manager-wide update and lifecycle announcements.

## 🔧 Troubleshooting

### RCON Connection Failures
The Web Manager runs inside a Docker container and communicates with your game servers via RCON.
* By default, it attempts to connect using `host.docker.internal`.
* If connections time out or fail, set the `RCON_HOST` environment variable in your `docker-compose.yaml` directly to your host's local network IP (e.g., `192.168.1.10`).

### SteamCMD update loops (`Missing configuration`)
If SteamCMD halts during automated updates claiming a missing configuration or returns `0x6` error codes:
* Ensure that no host mounts are masking `/home/pok/.steam/steam/steamapps/compatdata` inside the updater container. The Web Manager automatically runs updates using clean basic manifests to prevent this.
